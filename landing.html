常见问题

SkillScan 和普通 scanner 有什么区别？

Scanner 只负责「发现风险」。SkillScan 在 scanner 之上加了一层治理控制：准入、审批、复审、隔离、审计。企业买单的是控制权，不只是检测结果。

支持哪些类型的扩展？

当前先支持 Cursor skills。后续会扩展到 MCP、workflow packs 等可安装扩展。治理动作（准入、审批、复审、隔离、审计）是通用的。

如何集成到现有流程？

CLI 可在本地或 CI 中扫描，生成标准报告后提交到 SkillScan 服务端。服务端计算 verdict，命中高风险则进入审批流程。只有 Allowed 或 Approved 的版本才能通过发布门禁。

审批流程是怎样的？

审批绑定 repo@commit。版本一变，旧批准自动失效，需要重新审批。支持 needs_approval 标记，高风险内容必须人工批准后才能进入安装或发布路径。

FAQ

What's the difference between SkillScan and a scanner?

Scanners only detect risk. SkillScan adds a governance layer on top: admission, approval, re-review, isolation, and audit. Enterprises pay for control, not just detection.

What extension types does it support?

Currently Cursor skills. It will expand to MCP, workflow packs, and other installable artifacts. The governance actions are the same across artifact types.

How do I integrate with existing workflows?

The CLI scans locally or in CI and produces a standard report. Submit that to the SkillScan server, which computes the verdict. Risky versions require approval. Only Allowed or Approved versions pass release gates.

How does the approval workflow work?

Approval is tied to repo@commit. Version changes invalidate prior approvals. Content marked needs_approval must be manually approved before it can enter install or release paths.
