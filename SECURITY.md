# Security

## Reporting a vulnerability

If you believe you have found a security issue, please report it responsibly.

- **Email:** Prefer private disclosure to the maintainers. (Insert your security contact email here, e.g. `security@yourproject.com`.)
- **PGP:** (Optional) If you publish a PGP key, we can use it for encrypted reports.

Please include:

- Description of the issue and steps to reproduce.
- Impact (e.g. bypass of verdict, privilege escalation, data exposure).
- Any suggested fix or mitigation, if you have one.

**Do not** open a public GitHub issue for security-sensitive bugs.

## Response

- We will acknowledge receipt and aim to respond with an initial assessment in a reasonable time (best effort; we do not commit to a fixed SLA for community reports).
- We may ask for clarification or more details. We will work with you to understand and, if accepted, address the issue before public disclosure.
- We will credit you in advisories or release notes if you wish, unless you prefer to stay anonymous.

## Scope

- **In scope:** skillscan CLI (this repo), report/attestation schema and contract, and any documented API behavior used by the CLI.
- **Out of scope:** The hosted service (skillscan-cloud) has its own security process; contact the provider for that.
